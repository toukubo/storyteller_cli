# Story ネットワーク

「文章」は基本的に因果関係ネットワークの形状をとる。
因果関係ネットワークとは因果関係のネットワーク構造のこと。
因果関係ネットワークとは人工知能や脳の情報の基礎構造などで用いられるような情報構造。
また、ソフトウェア自体も「因果関係ネットワーク」構造を持つ。

ソフトウェアの外部仕様は画面などの機能操作を行うが、その操作シナリオは基本的に因果関係のパスを辿る。
例：多くの場合シナリオは、Aを作り、Aを閲覧し、Bを作り（その際にはAが必要。BはAに依存している）、Bを閲覧しと進む。
このとき「文章」は自然文章の形をとるが、「データモデル」がこのときA->Bと因果関係を持っている。また尤度（probability。発生する確率。頻度。スケール）がエッジ（因果関係）において発生している。A->Bが発生する確率と頻度の合算が尤度である。
A->Bは必ずa has many b (1:n）関係をとる。このように一般的なシステムは「因果関係」で「ERモデリング」されている。

またC->BでありかつA->Bであるケースがある。
ECプラットフォーム例などで「因果関係ネットワーク」としてのモデルと「シナリオ」の関係を説明する
```
ショップが登録する->ショップが商品を登録する→商品が公開される→商品が購入される
```
という「因果関係の筋」があり、また、
```
一般購入者ユーザーが登録する→ユーザーが商品を閲覧する→購入する
```
という筋がある。この一般ユーザーの登録はショップ登録の筋とは別に発生しつつ、因果関係ネットワーク上では「商品の購入や閲覧」で「合流」する。データのリレーションは因果の関係（因果関係）であり、またこの「データモデル」を図示したER図は、よく知られているように、「ネットワーク形状」をしている。因果関係ネットワーク形状をしている。

自然文でシナリオを書くと上記の筋は途中で、一度時系列（因果）関係をストップし、「一般ユーザー」に時間が遡る。

物語形式では「時間が必ず前から後に進む」形式をしており、これを逆転すると読者の可読性や没入性が下がる。複雑な物語では「ある主人子が登場して時間を進める」が「ある時点で別の登場人物の時間軸である過去に遡り」そして「ある時点でその登場人物たちが出会って時間軸を共有する」などがあるが、基本的にこれは「因果関係はネットワークで世界に存在し、これを一筋の陳述で説明する」ために図られた方法である。

システムソフトウェアを「シナリオ陳述」方式（ストーリー）で記載するときには、直接的に「データモデリング」をし始め、つまり調節的な「ソフトウェアの最終形式である精密な多次元因果関係ネットワーク」を記述するのではなく、画面を用いたシナリオや、「文章を用いたシナリオ」を記述することでなされるが、これで滞りなく陳述が最初から最後まで終えられているのであれば、因果関係ネットワークが含有されていると言っていい。

このとき「文章」と文章に出ている「名詞」の間に因果関係がある。


他方で、自然文章においては、特に英語言語などで「因果関係」を意味する文法や言語表現がある。人工知能研究先端ではすでに実戦に近い形で「文章と因果関係ネットワークの転換」やウェブをクロールして文章の中に現れる表現から言葉（名詞）と言葉の間の因果関係を取得する試みがあり、一定の成果をあげている。
人間の脳も文章を理解するときに全く同様のことを行っている（仮説であるが非常に多くのこのことを示唆する先行研究がある）。


Storytellerの発明者は人工知能の発明・研究・実装を行っているが、この「因果関係ネットワークと言語とゆうどによる汎用人工知能」を実装、実用している。特徴は「言語」にフォーカスした尤度と因果関係を記述しネットワークを教師付きで構成し、これにより「ある程度言語化されている情報については非常に精度の高い発案、思考、優先順位付け、意思決定を行う」ことにある。

すでに蓄積されている個別の因果関係から、「ネットワーク」を構築することができる。この「ネットワーク」と「自然文章から抽出したネットワーク」とを「ネットワーク比較」することで「正しいネットワークを高い精度で認識する」ことができる。

自然言語解析や認識の世界ではこれは王道的な方法であるが、これを「ソフトウェアの話法」に適応する。

つまりここでは「ソフトウェア仕様とモデルドメインで用いられた言葉」の両方のレイヤーにおいて「因果関係ネットワーク」が生成されるが、この一致を学習することで、「自然言語解析」を一般的な手法で行って生成される因果関係ネットワーク（特徴情報）と「storytellerを用いて開発されたモデルの関係性」とをネットワークマッチすることで、「自然言語からある程度モデル名称」とを解釈することができる。

シナリオ（ストーリー）に先んじて「会話などの、時系列には語られていない仕様」からも、ある程度「因果関係ネットワーク」を当てはめて「適格性の高いネットワーク」をマッチングレートの高い順から提案し「概ねこういうシナリオのシステムを作ろうとしているか」とシナリオ＝システムの提案を行うことが可能（仮説）

このとき、内部的にはモデリングが完了したデータのネットワーク（因果関係ネットワークであり、er図とモデル）も選定されている。
一般的なシステムに即して、crudなどやUIなども選定し、システムを生成し、実行することも可能であるはず。
